# Contributing Guidelines

## 📌 Notebook Requirements

Each notebook must include:

* Clear section title
* Explanation of concepts
* Code examples (with outputs)
* Comments explaining logic

---

## ✅ Naming Convention

```
section_X/notebooks/<your_name>.ipynb
```

---

## 🚫 Avoid

* Copy-pasting docs without understanding
* Broken code
* Overly long notebooks without structure

---

## 💡 Pro Tip

If your notebook can teach someone else → you're doing it right.
